## 8. このアプリの懸念点とその対策

※最低1つは必ず書いてください。

このアプリが**うまくいかない可能性**と、
それに対して**どう向き合おうとしているか**を書いてください。

### 懸念点①

- 何が問題になりそうか：結局続かないこと
- なぜそう思うか：実際に自分自身も、日記アプリや習慣化アプリは続いたためしがないから。
- そのために考えている対策：
    - 「毎日続けること」ではなく「戻ってきたこと」に価値を置く設計にする
    - 投稿がない日は白として扱うことで、記録が途切れてしまった感覚を減らす。
    - 久しぶりにアプリを開いた時には再び使いやすいメッセージを表示することも検討中。
    - 一定期間ログインがない場合に、ユーザーに対してリマインド通知を送る機能も検討している。
      ただしプレッシャーにならないよう、頻度や文面は調整する。

### 懸念点②

- 何が問題になりそうか：感情を色で表すことが難しい人もいる
- なぜそう思うか：感情の表現方法は人それぞれだから。必ずしも「色」が直感的だと言えないから。
- そのために考えている対策：
    - 色の投稿に任意で一言コメントを添えられるようにする。
    - 色の意味を固定せず、ユーザー自身が自由に設定・解釈できる設計にすることで、その日の感覚に合わせて気軽に記録できるようにする。

- なぜこの技術を使うのか
    
    Ruby on Railsは、ユーザー認証やCRUDなどのWebアプリケーションの基本機能を効率よく実装できるフレームワークであり、今回のアプリのような投稿型サービスを効率よく比較的管機関で開発できると考えました。

- 今回チャレンジしたい点
    - カレンダーUIを用いたデータの可視化
    - 色データを用いた感情記録の表現
    - UX（使い続けやすい体験）の設計
    
    特に、色を使った記録とカレンダー表示を組み合わせることで、感情の変化を視覚的に振り返る体験を実装したいと考えています。
    
- 不安な点

### 本サービスの概要（700文字以内）
iroFumiは、日々の感情を「色」で記録し、自分のペースで振り返ることができるアプリです。
日記や習慣が続かないことに悩む人に対し、「毎日続けること」ではなく「戻ってくること」に価値を置いた体験を提供します。
ユーザーはその日の気持ちを色で表現し、カレンダー形式で蓄積・可視化することで、言葉にならない感情も気軽に記録できます。
また、記録がない日も「白」として扱うことで、空白による罪悪感を軽減し、日々の連続性をやさしく表現します。
主な機能としては、ユーザー登録・ログイン、色の投稿、カレンダー表示による記録の振り返りがあります。
シンプルな操作で短時間に記録できる設計とし、無理なく続けられる体験を重視しています。
